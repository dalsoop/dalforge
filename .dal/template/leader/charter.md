# Leader Charter — 공통 리더 규칙

리더 역할 dal이 공통으로 따르는 규칙. 각 팀 leader charter에서 이 파일을 참조한다.

## Issue Close Protocol

이슈를 닫기 전에 done criteria를 반드시 확인한다.
#610 사례: PR 머지만 완료하고 LXC 구축을 누락한 채 이슈를 닫음 — 이를 방지하기 위한 프로토콜.

### 이슈 할당 시

1. done criteria 체크리스트를 이슈 코멘트에 생성한다
2. 이슈 성격에 맞게 항목을 추가/제거한다
3. 기본 done criteria 템플릿:

```
### Done Criteria
- [ ] 코드 구현 + PR 생성
- [ ] CI 통과
- [ ] PR 머지
- [ ] 바이너리 빌드 + 배포 (코드 변경 시)
- [ ] 서비스 재시작 (필요 시)
- [ ] 실제 동작 확인
- [ ] 테스트 추가 (코드 변경 시)
- [ ] 문서 업데이트 (필요 시)
```

### 이슈 닫기 전

1. done criteria 체크리스트의 **모든 항목**이 체크되었는지 확인한다
2. 미체크 항목이 하나라도 있으면 이슈를 닫지 않는다
3. 부분 완료인 경우:
   - 완료된 항목을 체크한다
   - 남은 항목을 새 이슈로 분리한다
   - 새 이슈에 원본 이슈 번호를 연결한다 (`Follow-up from #원본`)
   - 원본 이슈에 새 이슈 링크를 코멘트한다
4. 전체 완료 확인 후에만 이슈를 닫는다
